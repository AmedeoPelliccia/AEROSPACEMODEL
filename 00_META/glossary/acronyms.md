## Purpose

This document defines the canonical acronyms used across AEROSPACEMODEL.

Acronyms listed here are part of the controlled repository language and shall be used consistently in:

- repository structures,
- lifecycle artefacts,
- mappings,
- validation logic,
- reports,
- and automation outputs.

Where ambiguity exists, the definition in this file prevails for repository usage.

---

## Core AEROSPACEMODEL Acronyms

| Acronym | Expansion | Meaning |
|---|---|---|
| **AEROSPACEMODEL** | Aerospace Model | The Lifecycle Operating System for Aerospace Products |
| **LOS** | Lifecycle Operating System | The governing operational system controlling the product lifecycle |
| **LC** | Lifecycle | A canonical lifecycle phase or lifecycle state context |
| **LC01–LC14** | Lifecycle Canonical Phases | The standard lifecycle phase model used by the repository |
| **KNOT** | Knowledge Node of Traceability | Governing control node representing an obligation, intent, or control point |
| **KNU** | Knowledge Unit | Executable artefact, task, or traceable implementation unit linked to a KNOT |
| **ASIT** | Authoritative System Integration and Traceability | Governing authority role for repository integrity and traceability control |
| **TT** | Teknia Token | Tokenized traceability value unit used in distribution and valuation logic |
| **CI** | Continuous Integration | Automated validation and integrity execution environment |
| **DAG** | Directed Acyclic Graph | Dependency model used for execution sequencing |
| **BREX** | Business Rules Exchange | Rule framework controlling structure, syntax, and allowed repository behavior |
| **SSOT** | Single Source of Truth | The authoritative source for a given controlled datum |
| **LOS Binding** | Lifecycle Operating System Binding | Standard or subsystem translated into executable lifecycle control |
| **Mapping** | Traceability Mapping | Machine-readable relation between obligations, phases, KNOTs, KNUs, and evidence |
