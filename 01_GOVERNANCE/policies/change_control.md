**Authority:** ASIT  
**Status:** ACTIVE

---

## Purpose

Defines the process for controlling changes to governed objects.

## Rules

1. All changes to controlled objects require a formal CR.
2. CRs must follow the workflow defined in `01_GOVERNANCE/workflows/change_request_workflow.md`.
3. Changes to baselines require ASIT approval.
4. Emergency changes require post-hoc formal approval within 5 working days.
5. All approved changes must be reflected in `CHANGELOG.md`.
