## Purpose

Defines the digital thread model connecting all lifecycle artefacts.

## Thread Components

- Requirements (LC02)
- Architecture decisions (LC03)
- Design definitions (LC04)
- Analysis results (LC05)
- Verification records (LC06)
- Certification evidence (LC09)
- In-service data (LC11)
- Maintenance records (LC12)

## Invariant

Every controlled object must be reachable from its governing requirement via the thread.
